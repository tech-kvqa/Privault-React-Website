1. PROJECT OVERVIEW
--------------------------------------------------------------------------------
Privault is a premium React and TypeScript web application for an AI-powered
data privacy and security compliance management platform. The application acts 
as a high-fidelity control plane prototype showing operations across key risk
domains including:
- Consent Management & Auditing
- Breach Severity Classification & Response SLA timers
- DPIA (Data Protection Impact Assessment) Risk Matrix calculations
- DSR (Data Subject Request) identity validation and erasure logs
- Corporate Governance registers and DPO digital sign-offs
- Simulated Phishing campaigns and employee awareness training
- Third-party subprocessor vendor risk evaluation tables

The application is structured as a single-page marketing landing page with
interactive dashboard mockups representing each operational module.


2. TECH STACK
--------------------------------------------------------------------------------
The project leverages the following primary libraries and frameworks:
- React (v19.2) & React DOM: Client UI rendering.
- Vite: Lightning-fast developer server and bundling.
- TypeScript (v5): Static typing and strict interfaces.
- Tailwind CSS (v4): Design token configurations and responsive styling classes.
- Three.js (r185): WebGL core rendering of the padlock and key meshes.
- GSAP (GreenSock Animation Platform) & ScrollTrigger (v3.15): Scroll-scrubbing
  interpolation that binds user page scrolling to 3D object transforms.
- Framer Motion (v12): Fine-grained UI transitions (like dropdowns and sliders).
- Lucide React (v1.22): Scalable inline vector icon pack.
- React Router DOM (v6.22): Declarative browser history navigation.

4. KEY FILES & RESPONSIBILITIES
--------------------------------------------------------------------------------
- src/App.tsx
  Organizes the React routing framework and binds a listener to scroll the page
  to top (y: 0) on route modifications, preventing sticky offsets on page change.

- src/components/LockAnimation.tsx
  Lightweight React component representing the hero view. It feeds DOM element
  references to the 3D rendering hook and uses calculated progress boundaries to 
  fade content blocks (p1, p2, p3) in and out with smooth transitions.

- src/components/ComplianceEcosystem.tsx
  Immersive compliance solutions visualization. Integrates background particles, 
  SVG coordinate maps, quadratic Bezier calculations for line bending, and Framer 
  Motion springs to show interactive nodes. Hosts the "flower bloom" layout card.

- src/hooks/useLockScene.ts
  Hosts the core WebGL padlock engine. Instantiates the scene, lights, cameras,
  and handles mouse parallax updates via window mousemove listeners.

- src/constants/animation.ts
  Central repository for animation thresholds and responsive scaling values.


5. SCROLL-DRIVEN 3D LOCK ANIMATION DETAILS
--------------------------------------------------------------------------------
The scroll animation is executed inside `useLockScene.ts` by pinning the overall 
viewport wrapper in place while mapping the scroll progress decimal (0.0 to 1.0)
directly to a GSAP Scrub timeline:

Timeline Phase Checkpoints (defined in `constants/animation.ts`):
- Phase 1 (0.00 to 0.48):
  The key mesh begins suspended back in space (z: 2.6). As the scrollbar travels
  down, the key slides forward on the Z axis towards the lock body (z: 1.35).
- Phase 2 (0.48 to 0.74):
  The key continues moving forward on the Z axis (z: 0.445), inserting the teeth 
  fully into the chrome keyhole cylinder plug.
- Phase 3 (0.74 to 0.90):
  The key rotates 90 degrees clockwise (z rotation: Math.PI / 2) simulating a 
  physical cylinder release engagement.
- Phase 4 (0.90 to 1.00):
  The shackle mesh translates upwards on the Y axis (y: 0.38) with a spring bounce 
  ease ("back.out(2.5)") and rotates slightly on the Z axis (z: -0.12), mimic'ing
  the shackle latch swinging open. Concurrently, the point light glow (unlockGlow) 
  inside the keyhole rises in intensity (0 -> 25) alongside the keyhole rim's
  emissive glow, giving visual confirmation of the unlocked status.

Hinge Release & Pivot Mechanics:
The lock shackle is rendered relative to the parent lockGroup coordinate space. 
When the timeline hits 0.90, the shackle mesh position is moved upwards on the 
Y-axis. The rotation pivot is handled by placing the shackle tube mesh relative 
to the group center.

Mouse Parallax:
During each frame inside the requestAnimationFrame tick loop, the mouse position 
coordinates (tMX, tMY) calculated relative to the viewport center are smoothed 
using linear interpolation (lerp factor: 0.045). This smoothed coordinate offset 
is added directly to the camera's position on the X and Y axes, creating a 
premium 3D perspective depth effect.


6. COMPLIANCE ECOSYSTEM VISUALIZATION DETAILS
--------------------------------------------------------------------------------
The Compliance Ecosystem component replaces static cards with an SVG+HTML hybrid 
network. It uses several mathematical structures to create a fluid, premium feel:

- Space-Separated Marquee Header:
  An infinite scrolling marquee text ticker ("COMPLIANCE SOLUTIONS      SYSTEM...") 
  that slides right to left, utilizing spacious pre-formatted blank spaces to 
  separate compliance phrases cleanly instead of symbols. Scroll speed is 
  increased (16s duration) and padding reduced to py-4/py-6 to sit higher.

- Organic Moving Fog:
  Renders 3 large blurred backdrop divs in the background animating on the GPU in 
  slow loops to simulate a floating neon color fog utilizing the provided CSS hex 
  variables (#5e59d7, #494498, #9895eb).

- Full-Bleed Map Layout & Gaps:
  The visualizer container is borderless, marginless, and spans 100% full screen 
  width. Parent Solutions section padding is adjusted to pt-16 pb-0 to eliminate 
  the vertical gaps at the bottom. The visualizer sits directly on the bottom 
  section divider. The hero-to-solutions transition is updated to end at #1f3557, 
  removing the dark black fade band.

- Bottom Emergence & Shield Removal:
  The bottom central Core shield badge is removed completely, and fiber-optic 
  strands emerge directly from the bottom edge (500, 600) like a rising fountain.

- White Light Core Backdrop:
  A radial SVG gradient circle ("whiteLightGlow") centered at the bottom edge 
  (500, 600) which projects a soft white/lavender halo (#ffffff -> #dbdaf8 -> 
  transparent) behind the branches, creating a glowing dome effect.

- Stable Container & Strands:
  The overall map container, core, and SVG background strands remain strictly 
  stable on the grid. 

- Swaying Multi-Color Fibers always:
  120 fiber-optic lines radiate upward. Strands are colored always based on their 
  sector angle to form a glowing multi-color fan in the lavender palette (#5e59d7, 
  #c4c2f4, #aeacf0, #dbdaf8, #9895eb, #7d79e5) and sway organically:
  swayedAngle = angle + sin(time * speed + phase) * 0.022
  Strands bend/repel away from the cursor when within 120px proximity.

- Compact Node Link Badges (Icon + Title Only):
  The 7 compliance cards float independently. Each card's translateY offset 
  is animated inside the requestAnimationFrame loop to gently bob:
  floatY = sin(time * 2.2 + node.id) * 7
  The cards also tilt slightly in 3D parallax to the cursor.
  To prevent overlapping and double-card bugs, the cards are shrunk to a compact 
  width (w-[145px] p-3.5) containing only the product icon and title.
  Clicking a card routes the browser directly to /product/${node.slug} via 
  React Router Link. Hovering a card zooms it (scale: 1.15) and shoots responsive 
  laser pulses along its branch towards it.

- Mobile Carousel Redirects:
  Carousel swipe cards on mobile viewports also direct users to /product/${slug} 
  directly. All overlay modal panels and blurs have been removed.

   npm run build

   npm run preview


8. KNOWN QUIRKS OR GOTCHAS
--------------------------------------------------------------------------------
- GSAP Timeline Summation:
  The cumulative start offsets of the GSAP tweens must sum up exactly to 1.0. 
  Extending or shrinking phase timelines without adjusting the surrounding bounds 
  will cause scroll positions to map incorrectly or snap.
- Jittering Jumps:
  To prevent scrolling loops or page jumps from colliding with GSAP pinning, the 
  CSS declaration `scroll-behavior: smooth` is omitted on the HTML element.
- Responsive Layout Scale adjustments:
  If the viewport's aspect ratio falls below 0.85, the hook automatically
  reduces the model scale from 0.98 to 0.54 and repositions the lock coordinate 
  anchors, aligning the padlock model on top of the text block to prevent overlap 
  on narrow screens.
- Mobile Ecosystem Carousel:
  On viewports below 1024px, the network shifts to a mobile layout rendering a 
  `500x260` SVG tree on top of a swipeable card carousel. Swiping through the deck 
  highlights the respective branch and shoots glowing laser pulses up from the core.
